Section 01

Text Files

Human-readable files where data is stored as a sequence of characters, with each line ending in a newline character \n.

Key Characteristics

👁
Human Readable
Can be opened and edited in any plain text editor like Notepad or VS Code.
🔁
EOL Translation
Internal translation happens during read/write — the EOL character \n is handled automatically.
🔓
Less Secure
Generally considered less safe than binary files because they can be easily read and edited.

Opening & Closing Files

The open() function creates a file object that links Python to a physical file on disk.

Syntax
file_obj = open("filename.txt", "mode")
file_obj.close()   # Always close to free memory
💡 Best Practice — use with

The with statement automatically closes the file when the block ends, even if an exception occurs. Always prefer it over manual close().

Using the with clause
with open("data.txt", "r") as f:
    content = f.read()
# File is automatically closed here — no need for f.close()

File Access Modes

Mode Name Pointer Position If File Missing Key Behaviour
'r' Read Beginning Raises IOError Default mode. Read-only.
'w' Write Beginning Creates file Truncates (erases) all existing data immediately.
'a' Append End of file Creates file New data added after existing content — never overwrites.
'r+' Read & Write Beginning Raises IOError Both operations allowed; file must already exist.
'w+' Write & Read Beginning Creates file Truncates existing data; useful to overwrite then read back.
'a+' Append & Read End of file Creates file Pointer at end for writing; use seek(0) to read from start.
⚠ Watch Out

'w' and 'w+' truncate the file the moment they open it — all existing data is lost instantly, even before you write anything new.

Reading Functions

FunctionReadsReturnsCritical Nuance
read() Entire file String Includes every character — spaces, tabs, \n
read(n) n characters from current pointer String n counts spaces and \n as characters too
readline() One line String Includes the trailing \n
readline(n) n characters from current line String Stops at line boundary even if n is larger
readlines() All lines List of strings Each element retains its \n — strip before printing

Writing Functions

write(string)
  • Writes a single string to the file
  • Does not add \n automatically
  • You must include \n manually if needed
writelines(list)
  • Writes a list of strings
  • Does not add newlines between elements
  • Faster for bulk writes
Example — write vs writelines
with open("out.txt", "w") as f:
    f.write("Hello\n")              # Must add \n manually
    f.writelines(["Line1\n", "Line2\n"])  # \n must be in the list items

File modesr w a r+ w+ a+rb wb ab rb+ wb+ ab+

The pickle Module

To store or retrieve Python objects (lists, dicts, tuples, etc.) from a binary file, you use pickle.

🔽 Pickling (Serialization)
  • Converts a Python object → byte stream
  • Function: pickle.dump(data, file)
  • File must be opened in write binary mode: 'wb'
🔼 Unpickling (Deserialization)
  • Converts byte stream → Python object
  • Function: pickle.load(file)
  • File must be opened in read binary mode: 'rb'
Writing a binary file
import pickle

student = {"name": "Aanya", "roll": 42, "grade": "A"}

with open("student.dat", "wb") as f:
    pickle.dump(student, f)    # Serialise and save
Reading a binary file (handling EOF)
import pickle

def DisplayAll():
    with open("student.dat", "rb") as f:
        try:
            while True:
                R = pickle.load(f)   # Reads ONE object at a time
                print(R)
        except EOFError:            # Raised when file ends
            pass
🚨 Must Know — EOFError

Binary files with multiple pickled objects require a try…except EOFError loop. When pickle.load() reaches the end of the file, it raises EOFError — without catching it, your program will crash.

CRUD Operations on Binary Files

Search

Iterate through all records and compare the target field against each record.

Search by roll number
def Search(target):
    found = False
    with open("student.dat", "rb") as f:
        try:
            while True:
                R = pickle.load(f)
                if R["roll"] == target:
                    print("Found:", R)
                    found = True
        except EOFError:
            pass
    if not found:
        print("Record not found")
UpdateDelete

Since you can't edit a binary file in-place, you use the Temporary File Method:

1
Open original in rb and a new temp.dat in wb
2
Loop through all records using pickle.load()
3
Update: modify the matching record, then pickle.dump() to temp  |  Delete: simply skip the target record
4
Close both files
5
Use os.remove("original.dat") then os.rename("temp.dat", "original.dat")
Delete template
import pickle, os

def Delete(target_roll):
    records = []
    # Step 1: Load all records
    with open("student.dat", "rb") as f:
        try:
            while True:
                records.append(pickle.load(f))
        except EOFError:
            pass

    # Step 2: Write back — skipping the target
    with open("temp.dat", "wb") as f:
        for R in records:
            if R["roll"] != target_roll:
                pickle.dump(R, f)

    # Step 3: Replace original
    os.remove("student.dat")
    os.rename("temp.dat", "student.dat")

Section 03

CSV Files

Comma Separated Values — a plain-text format for tabular data, handled in Python via the built-in csv module.

Key Characteristics

📋
Tabular Format
Stores data in rows and columns — exactly like a spreadsheet — using plain ASCII text.
⚡
Lightweight
Smaller in size and faster to process compared to full spreadsheet formats.
🔁
Character Translation
Like text files, EOL translation occurs during read/write operations.
🔓
Less Secure
Plain text — easily opened and modified without special software.

Opening CSV Files — The Crucial Nuance

🚨 Critical — newline='' when Writing

When opening a CSV file for writing, always pass newline=''. Without it, Python inserts an extra blank line between every row because of how Windows handles line endings.

Writing
F = open("data.csv", 'w', newline='')
Reading
F = open("data.csv", 'r')

Writing to a CSV File

Create a writer object first, then use it to write rows.

FunctionInputWritesNuance
csv.writer(file, delimiter) File object Writer object Default delimiter is comma ','
writerow(list) A single List One row Each list element becomes one field
writerows(nested_list) List of lists Multiple rows at once Best for bulk/batch writing
Write example
import csv

data = [
    ["Name", "Roll", "Grade"],
    ["Aanya", 1, "A"],
    ["Rohan", 2, "B+"],
]

with open("students.csv", 'w', newline='') as f:
    writer = csv.writer(f)
    writer.writerow(data[0])       # Write header row
    writer.writerows(data[1:])     # Write all data rows at once

Reading from a CSV File

Create a reader object, then iterate over it with a for loop. Each row is returned as a List of strings.

⚠ Data Type Warning

CSV always returns data as strings. If you need to do arithmetic, convert first: int(row[1]), float(row[2]). Forgetting this is a very common exam mistake.

Read example with calculation
import csv

with open("marks.csv", 'r') as f:
    reader = csv.reader(f)
    for row in reader:
        print(row)                          # Each row is a List
        total = int(row[1]) + int(row[2])  # Must convert to int
        print("Total:", total)

Common Logic Patterns

TaskPattern
Count records Init counter = 0; increment inside the for loop
Search Check if row[i] == target inside the loop
Sum / Average Convert field: int(row[i]) or float(row[i]) before adding
Skip header Call next(reader) once before the loop, or use index slicing
